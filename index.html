Static CMOS Logic Design 静态 CMOS 逻辑设计

The core of static CMOS logic lies in using complementary PMOS pull-up networks (PUN) and NMOS pull-down networks (PDN) to implement Boolean functions. PUN and PDN are structurally dual to each other, ensuring that in stable states, one network is always conducting while the other is cut off. The tool below can help you visualize the transistor-level implementation of complex logic gates. 静态 CMOS 逻辑的核心在于使用互补的 PMOS 上拉网络（PUN）和 NMOS 下拉网络（PDN）来实现布尔函数。PUN 和 PDN 结构上互为对偶，确保在稳定状态下总有一个网络导通，另一个截止。下面的工具可以帮您可视化复杂逻辑门的晶体管级实现。

Circuit Delay Analysis and Optimization 电路延迟分析与优化

Circuit switching speed is a key performance metric. Logical Effort is a powerful model for analyzing and optimizing gate-level circuit delays. It decomposes delay into parasitic delay (p) related to gate characteristics and effort delay (f=gh) related to load. By adjusting gate sizes at each stage, the delay of the entire path can be minimized. 电路的开关速度是关键性能指标。逻辑努力（Logical Effort）是一种强大的模型，用于分析和优化门级电路的延迟。它将延迟分解为与门自身特性相关的寄生延迟（p）和与负载相关的努力延迟（f=gh）。通过调整各级门的尺寸，可以最小化整个路径的延迟。

Logical Effort Delay Visualization 逻辑努力延迟可视化

The normalized delay formula is $d = gh + p$. Where $g$ is logical effort, $h$ is electrical effort (load/input capacitance ratio), and $p$ is parasitic delay. Drag the sliders below to observe how changes in $g$ and $p$ affect the relationship between total delay and load. 归一化延迟公式为 $d = gh + p$。其中 $g$ 是逻辑努力，$h$ 是电气努力（负载/输入电容比），$p$ 是寄生延迟。拖动下面的滑块，观察 $g$ 和 $p$ 的变化如何影响总延迟与负载的关系。

Digital Circuit Testing 数字电路测试

Ensuring manufactured chips are defect-free is crucial. The Stuck-at Fault model is the most commonly used simplified fault model, which assumes that a line in the circuit is permanently stuck at logic '0' (SA0) or logic '1' (SA1). The goal of testing is to apply appropriate input vectors so that fault effects can be observed. 确保制造的芯片没有缺陷至关重要。固定型故障（Stuck-at Fault）模型是最常用的一种简化故障模型，它假设电路中的某条线永久地固定在了逻辑'0'（SA0）或逻辑'1'（SA1）。测试的目标就是施加合适的输入向量，使得故障效应可以被观测到。

This section deals with some more nuanced optimization problems that require deeper understanding beyond basic rules. Critical Path Optimization: Signal Arrival Order 关键路径优化：信号到达顺序

For a series NMOS pull-down network (such as a NAND gate), if one input signal arrives later than another, should the "slow signal" be placed on the transistor closer to the output or closer to ground? 对于一个串联的 NMOS 下拉网络 (如 NAND 门)，如果一个输入信号比另一个晚到，应该将这个"慢信号"放在靠近输出的晶体管上，还是靠近地的晶体管上？

Conclusion: Connecting the slowest (most critical) signal to the transistor closer to ground (A) provides better performance. 结论：将最慢（最关键）的信号连接到靠近地的晶体管 (A) 上性能更优。

Reason: 原因: This strategy leverages the "pre-discharge" effect. If the faster signal (B) arrives first, it turns on and begins discharging the capacitance of the internal node (X). When the slower signal (A) finally arrives, the voltage at its drain (X) is already lower, allowing transistor A to complete the conduction path to ground faster, thereby reducing the total delay contributed by the critical signal. 这种策略利用了"预放电"效应。如果速度较快的信号 (B) 先到达，它会导通并开始对内部节点 (X) 的电容进行放电。当较慢的信号 (A) 最终到达时，其漏极 (X) 的电压已经较低，使得晶体管 A 能够更快地完成整个路径到地的导通，从而减少了由关键信号贡献的总延迟。

Asymmetric Transistor Sizing 非对称晶体管尺寸

Sometimes, asymmetric transistor sizes are used to optimize specific paths, even when they are in the same series path. For example, in a series path, the transistor size for data signal A is 2, while the transistor size for reset signal Reset is 4/3. 有时，为了优化特定路径，会使用非对称的晶体管尺寸，即使它们在同一个串联路径中。例如，一个串联路径中，数据信号 A 的晶体管尺寸为 2，而复位信号 Reset 的晶体管尺寸为 4/3。

Purpose: Sacrifice the speed of non-critical paths to optimize the delay and power consumption of critical paths. 目的：牺牲非关键路径的速度，来优化关键路径的延迟和功耗。

Reason: 原因: The 'Reset' signal is typically not a performance-critical path. By using a smaller transistor (size 4/3), its input capacitance can be reduced, thereby lowering the load on the circuit driving the Reset signal. Although this increases the resistance of the Reset transistor itself, we can compensate by moderately increasing the transistor size of the critical data signal 'A' (size 2) to ensure the total pull-down resistance still meets design requirements. This is a typical engineering trade-off: sacrificing the speed of non-critical signals (Reset) in exchange for performance improvements and/or area/power reduction of critical signals (A). 'Reset' 信号通常不是性能关键路径。通过使用一个较小的晶体管 (尺寸 4/3)，可以减少其输入电容，从而降低驱动 Reset 信号的电路的负载。虽然这会增加 Reset 晶体管自身的电阻，但我们可以通过适度增大关键数据信号 'A' 的晶体管 (尺寸 2) 来补偿，以确保总下拉电阻仍然满足设计要求。这是一种典型的工程权衡：牺牲非关键信号（Reset）的速度，以换取关键信号（A）的性能提升和/或面积/功耗的降低。
